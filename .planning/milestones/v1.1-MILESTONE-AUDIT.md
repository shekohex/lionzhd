## Confirmed Cross-Phase Wiring

- `routes/web.php` routes `category-preferences.update/reset` into `CategoryPreferenceController`, which forwards `pinned_ids`, `visible_ids`, `hidden_ids`, and `ignored_ids` into `SaveUserCategoryPreferences`.
- `SaveUserCategoryPreferences` persists one user/media-type snapshot while preserving `pin_rank`, `sort_order`, `is_hidden`, and `is_ignored`, and `BuildPersonalizedCategorySidebar` reads the same snapshot back into shared browse/manage payloads.
- `VodStreamController` and `SeriesController` both consume personalized sidebar state, apply hidden/ignored filtering to listings, and emit `CategoryBrowseRecoveryStateData` for same-page recovery UX.
- `resources/js/hooks/use-category-browser.ts` uses one shared PATCH/DELETE contract for desktop/mobile preference saves, resets, and targeted unignore flows.
- `resources/js/components/category-sidebar/search.tsx` powers the same ranked category search model across desktop sidebar and mobile sheet without surfacing `All categories`.
- `SearchController`, `SearchMediaData`, and `resources/js/pages/search.tsx` keep `/search` URL-authoritative for `q`, `media_type`, `sort_by`, and pagination.
- `ResolveDetailPageCategories` feeds both movie and series show controllers, and `MediaHeroSection` renders clickable category chips that route back into browse pages.
- `tests/Browser/Support/browser-auth.php` is the shared live auth bootstrap reused by the milestone browser suites, and Phase 16 preserves one shared mixed-results `page` contract with history assertions gated by restored visible counts.
