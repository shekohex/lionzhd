In `AppServiceProvider::boot()`, define gates (names must be stable and used by middleware):
- `admin`: true when `user.role === admin`
- `super-admin`: true when `user.is_super_admin === true` (and role is admin)
- `member-internal`: true when role member + subtype internal
- `member-external`: true when role member + subtype external
- `server-download`: allowed for Admin only (members denied until Phase 2 ownership)
- `download-operations`: allowed for Admin only (pause/resume/cancel/retry/remove)
- `manage-users`: allowed for Admin (super-admin required for promote/demote in Plan 03)
- `auto-download-schedules`: allowed for Admin OR Internal Member; denied for External Member

- "Admin-only"
- "Internal-only"
- "External accounts cannot perform this action"

Update `TelescopeServiceProvider` and `PulseServiceProvider` gate definitions to be role-based:
- Replace `user->id === 1` with `user` is Admin.
