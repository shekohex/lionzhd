  <action>
Add `settings/users` endpoints protected by `can:admin`:
- `GET /settings/users` -> Inertia page `settings/users` with minimal list fields

Add endpoints for mutations:
- Toggle Member subtype (Internal/External): allowed for any Admin
- Promote/demote Admin: allowed for super-admin only (use `can:super-admin` middleware or `Gate::authorize('super-admin')` so deny reason flows into Inertia 403 UX)
- Transfer super-admin: allowed for current super-admin only (same: `can:super-admin` / `Gate::authorize`)

Hard rules (server-enforced):
- Super-admin cannot be demoted or deleted via these endpoints.
- System must not allow a zero-admin state: prevent demoting the last Admin.
- When transferring super-admin: ensure exactly one user has `is_super_admin=true` after operation.

Keep payloads simple (request validation). Do not add new packages.
