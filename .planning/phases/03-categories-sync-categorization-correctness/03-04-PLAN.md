  <action>
Add admin-only settings routes (under existing `can:admin` group):
- GET `settings/synccategories` -> `SyncCategoriesController@edit` (route name: `synccategories.edit`)
- PATCH `settings/synccategories` -> `SyncCategoriesController@update` (route name: `synccategories.update`)
- GET `settings/synccategories/history` -> `CategorySyncRunsController@index` (route name: `synccategories.history`)

Controller behaviors:

`SyncCategoriesController@edit`:
- Render Inertia page `settings/synccategories`.

`SyncCategoriesController@update`:
- Preflight fetch category lists (VOD+Series) using Xtream connector + category requests.
- If either selected source returns zero categories AND request is not forced:
  - Return back with a user-facing warning (validation error or flash) that requires explicit confirmation.
  - UI will re-submit with per-source force flags (e.g. `forceEmptyVod=1`, `forceEmptySeries=1`) based on which source(s) were empty.
- If forced or non-empty:
  - Dispatch the `App\Jobs\SyncCategories` job (do not block), passing force flags so the job/action can safely apply empty-source runs only when confirmed.
  - Return back with success message.

`CategorySyncRunsController@index`:
- Query recent `CategorySyncRun` rows (latest first) and return Inertia page `settings/synccategories-history` with runs + pagination.

Do not mention Live categories in UX.

Add Feature tests for controller-level confirmation behavior in `SyncCategoriesControllerTest`:
- Mock Xtream category preflight where VOD or Series returns zero categories.
- First PATCH to `synccategories.update` without force should not dispatch job and should return warning/validation requiring confirmation.
- Second PATCH with per-source force flag(s) should dispatch `SyncCategories` job.
- Add one happy-path test where non-empty sources dispatch immediately.
  </action>
