<task type="auto">
  <name>Task 2: Add BuildCategorySidebarItems action (A–Z, Uncategorized last, disabled-zero via aggregate counts)</name>
  <files>app/Actions/BuildCategorySidebarItems.php</files>
  <action>
- Create `BuildCategorySidebarItems` action using existing `AsAction` pattern (callable via `::run(...)`).
- Inputs: `MediaType $mediaType`, `?string $selectedCategoryId`.
- For the requested media type:
  - Load the categories dataset via `Category::query()` scoped by the correct flag (`in_vod` for movies, `in_series` for series).
  - Compute per-category counts via ONE aggregate `GROUP BY category_id` query on the correct media table.
  - Compute uncategorized count with a dedicated query that counts rows where `category_id` is null/empty-string OR equals the correct `Category::UNCATEGORIZED_*_PROVIDER_ID`.
- Produce a list of `CategorySidebarItemData` items with:
  - Ordering: A–Z by name (case-insensitive), with Uncategorized forced to bottom.
  - `isUncategorized: true` only for the system uncategorized provider-id for that media type.
  - `disabled: true` only when `count === 0` AND the item is NOT the currently-selected category.
- Do NOT include an "All categories" item here (frontend adds it).
- Ensure the implementation does not do per-category item-count queries (no N+1).
  </action>
  <verify>php artisan test --filter SyncCategoriesControllerTest</verify>
  <done>
- Action returns `CategorySidebarItemData[]` for Movies or Series.
- Uncategorized item is last, and zero-item categories are disabled unless selected.
- Only aggregate queries are used for counts.
  </done>
</task>
