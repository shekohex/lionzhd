  <name>Task 1: Add validated `?category=` filtering + sidebar props to Movies index controller</name>
  <files>app/Http/Controllers/VodStream/VodStreamController.php</files>
  <action>
- Update `index()` to accept `Request $request` and return `Response|RedirectResponse`.
- Read `?category=` from the query string; treat blank/whitespace as null (All).
- Validate non-null category IDs against the Movies dataset:
  - Allowed IDs are `Category::query()->where('in_vod', true)->pluck('provider_id')`.
  - If invalid: `return to_route('movies')->with('warning', 'Category not found. Showing all categories.');`
- Apply category filtering to the Movies query:
  - Base: existing watchlist `withExists` + `orderByDesc('added')`.
  - If selected category is `Category::UNCATEGORIZED_VOD_PROVIDER_ID`: filter where `category_id` is null OR '' OR equals the uncategorized provider-id.
  - Else: filter `where('category_id', $categoryId)`.
- Paginate with `->paginate(20)->withQueryString()`.
- Return Inertia `movies/index` with these props (use closures so partial reloads can request only subsets):
  - `movies`
  - `filters` as `CategoryBrowseFiltersData` with `category: $categoryId`
  - `categories` as `BuildCategorySidebarItems::run(MediaType::Movie, $categoryId)`
  </action>
  <verify>php artisan test --filter MoviesCategoryBrowseTest</verify>
  <done>
- `/movies` renders `movies/index` with `movies`, `filters.category`, and `categories` props.

- Pagination links keep `category=` when active.
  </done>
