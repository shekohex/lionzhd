<task type="auto">
  <name>Task 1: Add validated `?category=` filtering + sidebar props to Series index controller</name>
  <files>app/Http/Controllers/Series/SeriesController.php</files>
  <action>
- Update `index()` to accept `Request $request` and return `Response|RedirectResponse`.
- Read `?category=` from the query string; treat blank/whitespace as null (All).
- Validate non-null category IDs against the Series dataset:
  - Allowed IDs are `Category::query()->where('in_series', true)->pluck('provider_id')`.
  - If invalid: `return to_route('series')->with('warning', 'Category not found. Showing all categories.');`
- Apply category filtering to the Series query:
  - Base: existing watchlist `withExists` + `orderByDesc('last_modified')`.
  - If selected category is `Category::UNCATEGORIZED_SERIES_PROVIDER_ID`: filter where `category_id` is null OR '' OR equals the uncategorized provider-id.
  - Else: filter `where('category_id', $categoryId)`.
- Paginate with `->paginate(20)->withQueryString()`.
- Return Inertia `series/index` with these props (use closures so partial reloads can request only subsets):
  - `series`
  - `filters` as `CategoryBrowseFiltersData` with `category: $categoryId`
  - `categories` as `BuildCategorySidebarItems::run(MediaType::Series, $categoryId)`
  </action>
  <verify>php artisan test --filter SeriesCategoryBrowseTest</verify>
  <done>
- `/series` renders `series/index` with `series`, `filters.category`, and `categories` props.
- Invalid `?category=` redirects to `/series` and flashes `warning`.
- Pagination links keep `category=` when active.
  </done>
