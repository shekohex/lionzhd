  <name>Task 2: Implement persisted cancel + sticky pause intent</name>
  <files>

app/Http/Controllers/MediaDownloadsController.php

  <action>
- Add a configured allowlist root for any server-side file deletion:
  - In `config/services.php` under `aria2`, add `download_root`:
    - Env: `ARIA2_DOWNLOAD_ROOT`
    - Default: `storage_path('app/public/downloads')`

- Add `app/Actions/Downloads/DeleteDownloadFiles.php`:
  - Input: list of file paths.
  - Determine allowlisted root via `config('services.aria2.download_root')`.
  - Delete only files whose `realpath()` is inside the `realpath($allowlistedRoot)` directory.
  - For each file path, also attempt to delete `${path}.aria2` when it exists and is within root.
  - If a candidate path does not exist (`realpath` false), skip (do not attempt unsafe deletes).
  - Add a negative-path guard: if the provided aria2 `dir` (from `tellStatus`) is NOT inside the allowlisted root, skip deletion and return a user-facing error (or log + surface an error) rather than attempting any delete.

- Add `app/Actions/Downloads/CancelDownload.php`:
  - Input: `MediaDownloadRef $download`, `bool $deletePartial`.
  - If `canceled_at` already set: no-op (idempotent).
  - Snapshot `tellStatus` (keys include `dir,files,errorCode,errorMessage`) and persist:
    - `download_files` as list of `files[*].path` (and keep `cancel_delete_partial`).
  - Stop aria2 download using `ForceRemoveRequest`:
    - If the RPC call fails, return an error and do NOT set `canceled_at`.
  - Persist `canceled_at=now()` only after a successful ForceRemove.
  - If `$deletePartial === true`, call `DeleteDownloadFiles` using the snapshot paths, but ONLY after validating `tellStatus.dir` is within the configured allowlisted download root.

- Extend `EditMediaDownloadData` to accept `delete_partial?: bool` (default false).

- Update `MediaDownloadsController`:
  - Treat DELETE `/downloads/{model}` (`destroy`) as the Cancel/Abort operation:
    - Call `CancelDownload` with `deletePartial=false`.
    - Do NOT delete the DB row.
  - For PATCH `/downloads/{model}` (`edit`):
    - Guard: if `canceled_at` is set, reject all actions with a user-facing error.
    - On `pause`: set `desired_paused=true` after successful aria2 pause.
    - On `resume`: set `desired_paused=false` after successful aria2 unpause.
    - On `cancel`: call `CancelDownload` with `$payload->delete_partial`.
    - Keep existing `remove` behavior for non-canceled rows.
  </action>
  <verify>php artisan test</verify>
  <done>
- Cancel results in persisted terminal cancel state and does not delete the row.
- Pause/resume toggles persisted `desired_paused` intent.
  </done>
</task>
