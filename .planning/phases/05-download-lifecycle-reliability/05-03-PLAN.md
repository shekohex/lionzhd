  <action>
- Add `ClassifyDownloadFailure`:
  - Input: aria2 `errorCode` (int) + `errorMessage` (?string).
  - Output: classification including `isTransient`.
  - Treat transient by default for: network/timeouts and upstream/provider 5xx:
    - `errorCode` in {2,6,19,29} is transient.
    - Best-effort: treat messages containing an HTTP 5xx as transient.

- Add `ComputeRetryBackoff`:
  - Input: attempt number (1..5).
  - Output: bounded exponential backoff seconds (cap).
  - Use a deterministic schedule (so tests can assert exact values):
    - base: 5 seconds
    - formula: `min(5 * (2 ** (attempt - 1)), 300)` (cap at 5 minutes)

- Add `MonitorDownloads` job (ShouldQueue + ShouldBeUnique):
  - Runs every minute.
  - Selects non-terminal downloads (`canceled_at` null) and hydrates aria2 status with keys:
    `gid,status,errorCode,errorMessage,dir,files,totalLength,completedLength,downloadSpeed`.
  - Persist file targets for later restart-from-0 determinism:
    - When `download_files` is null and `tellStatus.files[*].path` is present, persist `download_files` snapshot.
  - Persists latest `last_error_code/last_error_message` when aria2 status is `error`.
  - When status is `error` and classification is transient:
    - Increment `retry_attempt` (max 5).
    - Compute `retry_next_at = now() + backoff`.
    - Dispatch `RetryDownload` job delayed until `retry_next_at`.
  - When attempts exceed 5: stop scheduling further auto-retries (leave failed state with manual retry available).
  - Enforce sticky pause:
    - If `desired_paused=true` and aria2 status is `active|waiting`, call aria2 pause.
    - Never auto-unpause.

- Add `RetryDownload` job that performs the retry at `retry_next_at`:
  - Guard: `canceled_at` is null, `desired_paused` is false, and now >= `retry_next_at`.
  - Best-effort cleanup: `removeDownloadResult(old_gid)`.
  - Re-add download via aria2 `addUri` by reconstructing the Xtream download URL from the row:
    - VOD: fetch vod info + build URL + out
    - Series episode: fetch series info + locate episode + build URL + out
  - Ensure resume works (RELY-03):
    - Use the SAME `out` path as initial downloads (computed via `CreateDownloadOut`), so aria2 can resume using the existing control file.
    - Ensure `continue=true` and `auto-file-renaming=false` remain in the aria2 options.
  - Update the row's `gid` to the new gid.
  - On success: clear `retry_next_at`.

- Wire scheduler in `routes/console.php`:
  - `Schedule::job(MonitorDownloads::class)->name('monitor-downloads')->description('Monitor aria2 downloads and schedule retries')->everyMinute()->withoutOverlapping()->sentryMonitor()`.

- Make app-managed retry policy authoritative by reducing aria2 internal retries:
  - In `DownloadMedia` + `BatchDownloadMedia`, set `max-tries` to `1` and `retry-wait` to `0` (keep `continue=true`).
  </action>
