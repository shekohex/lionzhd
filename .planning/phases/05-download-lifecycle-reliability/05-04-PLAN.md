<task type="auto">
  <name>Task 2: Implement Phase-05 download row UX (progress, cancel, retry, placeholders)</name>
  <files>resources/js/components/download-info.tsx</files>
  <action>
- Update progress rendering (locked decisions):
  - Show percent + bytes + speed when total size is known.
  - Percent precision: 1 decimal.
  - Clamp invalid values to a sane range (0..100 for percent).
  - If total size is unknown/unreliable (totalLength <= 0): show indeterminate progress + bytes (and speed if available), no percent.
  - Hide speed when 0/unavailable.

- Hydration failures / missing status:
  - When `download.downloadStatus` is missing (or cannot be hydrated), show shimmer/skeleton placeholders for numeric values (do not show last-known numbers).
  - On initial start (no reliable numbers yet), show "Starting...".

- Cancel UX (locked decisions):
  - Replace `window.confirm` cancel with a dialog that always confirms.
  - Dialog includes a "Delete partial data" checkbox (default unchecked).
  - On confirm, call PATCH `downloads.edit` with `{ action: 'cancel', delete_partial: <checkbox> }`.

- Terminal canceled behavior:
  - If `download.canceled_at` is set: show status "Canceled" and render NO further actions.

- Retry UX (locked decisions):
  - When `download.retry_next_at` is set in the future: display status like "Retrying in Xm" with attempt count.
  - During cooldown/backoff: disable retry and show a countdown.
  - For terminal failed (error without pending retry): show failure reason inline.
    - Members: friendly summary.
    - Admins: expandable technical details.
  - Wire retry action payload explicitly:
    - Retry button triggers `router.patch(route('downloads.edit', { model: download.id }), { action: 'retry' }, { preserveScroll: true, preserveUrl: true })`.
    - If offering a restart-from-0 choice in the UI prompt, send `{ action: 'retry', restart_from_zero: true }`.
  </action>
  <verify>corepack pnpm -s run lint && corepack pnpm -s run build</verify>
  <done>
- UI matches Phase 05 context decisions for progress, cancel, retry/cooldown, and failure messaging.
  </done>
</task>
