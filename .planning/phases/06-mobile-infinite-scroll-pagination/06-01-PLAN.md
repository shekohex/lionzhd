  <action>
- Update Movies browse query (VodStreamController@index):
  - Ensure a stable total ordering by adding a unique secondary order:
    - `orderByDesc('added')` then `orderByDesc('stream_id')`.
  - Introduce snapshot-consistent pagination using two query params:
    - `as_of` (timestamp/string cutoff against `vod_streams.added`)
    - `as_of_id` (tie-break cutoff against `vod_streams.stream_id`)
  - Cutoff behavior (prevents shifts even if new inserts share the same timestamp):
    - If request includes `as_of` + `as_of_id`, constrain to:
      - `added < as_of` OR (`added = as_of` AND `stream_id <= as_of_id`).
    - If request lacks them, compute both from the newest row in the ordered result-set (first row after applying filters) and treat that as the snapshot cutoff.
  - Append both into paginator URLs so `next_page_url` includes them:
    - `->appends(['as_of' => $asOf, 'as_of_id' => $asOfId])->withQueryString()` (ensure category remains preserved).

- Update Series browse query (SeriesController@index):
  - Stable total ordering:
    - `orderByDesc('last_modified')` then `orderByDesc('series_id')`.
  - Same snapshot behavior using the same query params:
    - `as_of` (cutoff against `series.last_modified`)
    - `as_of_id` (cutoff against `series.series_id`)
  - Cutoff must be NULL-safe because `series.last_modified` is nullable:
    - Always include NULL `last_modified` rows (they should remain reachable/paginated).
    - Apply cutoff for non-NULL rows using:
      - `last_modified < as_of` OR (`last_modified = as_of` AND `series_id <= as_of_id`).
    - If request lacks `as_of` + `as_of_id`, compute both from the newest row in the ordered result-set after filters.
  - Append both params into paginator URLs like Movies.

Constraints:
- Do not change desktop pagination UX; this is purely the pagination contract.
- Keep existing category filtering logic unchanged.
  </action>
