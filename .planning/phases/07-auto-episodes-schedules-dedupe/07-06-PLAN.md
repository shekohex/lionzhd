<task type="auto">
  <name>Implement scan+diff+cap pipeline with run/event logging and next_run_at updates</name>
  <files>

app/Models/AutoEpisodes/SeriesMonitorEpisode.php

  <action>
    - Implement `ScanSeriesForNewEpisodes` action (called by `RunMonitorScan`) to:
      - Load monitor (with user/series).
      - Create a `SeriesMonitorRun` row (status running, trigger derived from job context).
      - Determine `window_start_at = monitor.last_successful_check_at` (nullable) and `window_end_at = now()`.
      - Fetch Xtream series info via `GetSeriesInfoRequest(series_id)` and iterate episodes.
      - Filter episodes by `monitor.monitored_seasons` (if empty: treat as "all seasons").
      - For each episode, upsert/ensure a `SeriesMonitorEpisode` exists (unique by monitor+episode_id) and update `last_seen_at`.
        - Default state rules:
          - New rows default to `pending`.
          - Existing rows keep their current state unless transitioned explicitly.
      - Baseline rule (prevents full-history auto-backfill):
        - If `monitor.last_successful_check_at` is null AND run trigger is NOT `backfill`:
          - Mark ALL discovered episodes as `skipped` (or equivalent “known, do not queue”).
          - Do NOT queue any episode during this run.
          - Set `monitor.last_successful_check_at = window_end_at` and compute/persist `next_run_at`.
          - Record a run status indicating baseline (use existing status enum; keep events minimal).
          - Return success.
      - Compute queue candidates as episodes with state in `{pending, failed}` (exclude `skipped`, `queued`, `downloaded`, `canceled`).
      - Sort oldest-first (season asc, episode_num asc).
      - Apply `per_run_cap`:
        - First N: attempt queue via `QueueEpisodeDownload`.
        - Remainder: keep as pending and record `deferred` events.
      - For each queued attempt:
        - If queued: update episode row state + `last_queued_at` + `last_download_ref_id`.
        - If duplicate: keep episode state stable; create `duplicate` event.
        - If error (e.g. unsafe episode id): keep episode as pending; create `error` event with episode id metadata.
      - On success:
        - Set monitor `last_successful_check_at = window_end_at`.
      - Always:
        - Set monitor `last_attempt_at`, `last_attempt_status`.
        - Compute and persist `next_run_at` using `ComputeNextRunAt`.
        - Finalize run row: status + counts + finished_at.
      - On failure/exception:
        - Persist run status `failed` + error_message; leave `last_successful_check_at` unchanged; still advance `next_run_at`.
    - Update `RunMonitorScan` job to:
      - accept a trigger enum (`scheduled|manual|backfill`) and optional options (e.g. `backfill_count`)
      - pass trigger/options to `ScanSeriesForNewEpisodes`
      - ensure idempotency under the monitor lock.
  </action>
  <verify>php artisan test --filter MonitorScanDedupeTest</verify>
  <done>A scan produces run+events, queues up to cap, defers remainder, advances next_run_at, and never duplicates queue rows</done>
</task>
