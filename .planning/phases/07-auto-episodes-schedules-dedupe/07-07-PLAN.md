  <name>Add monitor management controllers + routes (series endpoints + settings page)</name>
  <files>

app/Http/Controllers/AutoEpisodes/SeriesMonitoringRunNowController.php
app/Http/Controllers/AutoEpisodes/AutoEpisodesPauseController.php
  </files>
  <action>
    - Implement endpoints:
      - Series detail actions (in `routes/web.php`, under auth+verified):
        - `POST /series/{model}/monitoring` → enable/create (requires schedule + seasons)
        - `PATCH /series/{model}/monitoring` → edit schedule/seasons
        - `DELETE /series/{model}/monitoring` → disable-only OR disable+remove-from-watchlist (boolean payload)
        - `POST /series/{model}/monitoring/run-now` → run now (cooldown enforced)
      - Settings management page:
        - Replace the current closure for GET `settings/schedules` with `MonitoringPageController@index` returning Inertia `settings/schedules` with props.
        - Add mutation routes under `Route::middleware('can:auto-download-schedules')` for:
          - bulk preset apply (list of series ids + schedule preset)
          - global pause/resume (toggle user.auto_episodes_paused_at)
    - Access control:
      - Apply `can:auto-download-schedules` middleware ONLY to mutation routes (not GET pages).
      - For series endpoints, apply `can:auto-download-schedules` as middleware.
    - Controllers behavior:
      - Enabling monitoring must validate series is in the current user's watchlist (otherwise error).
      - When enabling or updating schedule, compute and persist `next_run_at` via `ComputeNextRunAt` but do NOT auto-run.
      - Run-now dispatches `RunMonitorScan` and updates `run_now_available_at` based on config cooldown.
      - Disable+remove should delete watchlist entry (and rely on FK cascade to remove monitor if configured).
  </action>
  <verify>php artisan test --filter AutoEpisodesAccessTest</verify>
  <done>Routes exist, mutations are gated, and controllers enforce watchlist + schedule requirements</done>
</task>
